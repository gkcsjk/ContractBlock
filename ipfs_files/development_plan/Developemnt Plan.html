Blockchain Contract Platform Development

Main idea:

Ethereum DApp (Decentralized Application) Development

System Structure:

Platforms and languages:

Web:

HTML, CSS, JavaScript, NodeJS, Express, React

Blockchain:

Soliity, Remix, geth...

Data:

IPFS, MongoDB

Functions:

Two or more parties can reach agreements (contracts) and write them into the block chain, which are permanent and unchangeable.

  • User registration
  • Mining or purchasing Ether
  • Generate, view, delete, modify contract off chain and upload/synchronize contracts to blockchain (consuming Ether)
  • Transfer Ether between users
  • Blocks explorer (for demo)

Notes:

Always save big files to IPFS. IPFS will return its unique Hash. Using this Hash in IPFS distributed web can access the related data. It can reduce the data written to blockchain, which will consume Ether.

Writing data to blockchain will consume Ether (Gas). Therefore, users need to have Ether in their account to make sure they can support their data writing. Reading data from blockchain does not consume Ether.

Users can get Ether from mining or transferring from other accounts. In the real chain, mining can be very difficult and consume a lot of system resources. But in this project, we may build a private chain which makes mining kind of easy.
